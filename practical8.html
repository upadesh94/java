Practical 8: Exception Handling

Implementation of Exception Handling in Java with Multiple Examples

Problem Statement

Implementation of Exception Handling in Java with Multiple Examples demonstrating different types of exceptions and their handling.

Example 1: Multiple Exception Handling

This example demonstrates handling multiple specific exceptions:

  • ArithmeticException for division by zero
  • ArrayIndexOutOfBoundsException for array access
  • General Exception handling

import java.util.*;

public class exceptionex1 {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int[] arr = new int[2];
        try {
            System.out.println("Enter the two numbers");
            arr[0] = sc.nextInt();
            arr[1] = sc.nextInt();
            int ans = arr[1] / arr[0];
            System.out.println("Division of two numbers is " + ans);
        } catch (ArithmeticException e) {
            System.out.println("Cannot divide by zero");
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Array index out of bound");
        } catch (Exception e) {
            System.out.println("Invalid input");
        }
    }
}

Example 2: Input Mismatch Exception

This example demonstrates handling input mismatch exceptions:

  • InputMismatchException for invalid input types
  • Using getMessage() method
  • Proper resource cleanup with Scanner

import java.util.*;

public class exceptionex2 {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int x, y;

        try {
            System.out.println("Enter the first number:");
            x = sc.nextInt();
            System.out.println("Enter the second number:");
            y = sc.nextInt();
            System.out.println("First number is: " + x + " and second number is: " + y);
        } catch (InputMismatchException e) {
            System.out.println("Invalid input: " + e.getMessage());
        }

        sc.close();
    }
}

Example 3: General Exception Handling

This example demonstrates handling all exceptions with a single catch block:

  • Using general Exception class
  • Handling multiple types of exceptions
  • Simple error message display

import java.util.*;

public class exceptionex_3 {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int[] arr = new int[2];
        try {
            System.out.println("Enter the two numbers");
            arr[0] = sc.nextInt();
            arr[1] = sc.nextInt();
            int ans = arr[1] / arr[0];
            System.out.println("Division of two numbers is " + ans);
        } catch (Exception e) {
            System.out.println("Any other unexpected exceptions.");
        }
    }
}

How it works

  1. Exception Types:
    • ArithmeticException: Division by zero
    • ArrayIndexOutOfBoundsException: Invalid array access
    • InputMismatchException: Invalid input type
    • General Exception: Catch-all for unexpected errors
  2. Exception Handling Features:
    • Multiple catch blocks for specific exceptions
    • Exception message retrieval
    • Resource cleanup (Scanner)
    • User-friendly error messages
  3. Best Practices:
    • Specific exceptions before general ones
    • Proper resource management
    • Clear error messages
    • Graceful error handling
